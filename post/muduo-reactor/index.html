Muduo Reactor

2021-09-24 3 min read # muduo
  • 现代C+的Reactor模式实现
    1. EventLoop：事件循环
    2. Poller：IO multiplexing，获取IO事件
    3. Channel：事件分发，将IO multiplexing的IO事件分发给对应的文件描述符的事件处理函数

EventLoop

  • EventLoop会进行loop循环，loop中包含IO multiplexing的阻塞（select,poll,epoll等）
  • 一个EventLoop只属于一个线程，即在该线程中进行循环。
  • 一个线程也只能有一个EventLoop。（这一点是通过thread_local变量来判断的）
  • EventLoop的接口分两种，一种是线程安全的（可以跨线程调用），另一种是必须在当前线程调用的。
    判断方法：
  1. 非线程安全函数：（assertInLoopThread）在EventLoop构造时会记录构造该对象的tid。在运行EventLoop的非线程安全函数时会判断tid是否和std::this_thread::get_id()相同，不相同的话说明在其他线程运行该EventLoop，会abort。
  2. 线程安全函数：核心原理是通过加锁mutex来实现线程安全。

Channel

  • 每个管道只负责一个文件描述符的事件分发
  • 事件分发通过回调，即将std::function<void()>（通常是通过std::bind(...)构造的)设置为Channel的回调函数成员，比如readCallback,writeCallback等
  • Channel和Poller的交互：
  1. 要允许读或者写的话，是将一个变量int events进行或操作，如读就是设置event |= kReadEvent
  2. Poller捕获到事件的话，就通过设置Channel的int revents来通知Channel什么事件发生（读、写事件等）

  • 监听事件
    如果是可读，则设置events|=(POLLIN|POLLPRI)
    如果是可写，则设置events|=(POLLOUT)

  • 事件分发

  1. 写事件：POLLOUT
  2. 读事件：POLLIN,POLLPRI,POLLRDHUP
    其中POLLIN表示有普通数据可读，POLLPRI表示高优先级数据可读（如TCP socket的带外数据可读），POLLRDHUP表示对端socket使用close或者shutdown write
  3. 错误事件：POLLERR,POLLNVAL
    POLLERR表示错误情况，POLLNVAL表示文件描述符没有open。
  4. 关闭事件：POLLHUP
    对端socket关闭。
  • 对于读处理：
    通常需要写读取以后，判断读取的长度
  1. 大于0是正常读取消息。
  2. 等于0表示对方关闭。
  3. 小于0则为读取错误。
  • 对于写处理：
    当读取到对方关闭close时（无论是POLLHUP还是从read中读取到的消息长度为0），都需要及时close socket，无论输出缓冲区是否有内容。
    只有在主动shutodwn时才考虑先输出缓冲区的内容再延迟shutdown
